RLD depends on two price concepts:

| Price | Meaning | Typical source |
| --- | --- | --- |
| Index price | Fundamental value of `wRLP` from the underlying borrow rate. | `IRLDOracle`, such as `RLDAaveOracle`. |
| Mark price | Tradable pool price of `wRLP` against collateral. | `ISpotOracle`, such as `GhostSingletonOracle`. |

The index anchors the derivative. The mark drives funding and user execution.

## Index Price

For Aave-style markets, `RLDAaveOracle` reads `currentVariableBorrowRate` from the Aave pool reserve data and maps it into WAD price units:

```text
P_index = currentVariableBorrowRate * K / 1e9
```

The division converts Aave RAY precision to WAD precision after multiplying by `K = 100`.

The implementation applies:

| Bound | Value |
| --- | --- |
| Rate cap | 100% in RAY (`1e27`). |
| Minimum price | `0.0001` in WAD (`1e14`). |

The paper also proposes TWAR, utilization-vs-IRM validation, and future ZK storage proofs. The current `RLDAaveOracle` is the on-chain Aave adapter for the rate-to-price mapping and cap/floor behavior.

## Standard Funding

`StandardFundingModel` calculates funding from mark-index divergence:

```text
normalizedMark = markPrice / currentNormalizationFactor
fundingRate = (normalizedMark - indexPrice) / indexPrice
exponent = -fundingRate * dt / fundingPeriod
NF_new = NF_old * exp(exponent)
```

Interpretation:

| Condition | NF effect | Economic result |
| --- | --- | --- |
| `mark > index` | NF decreases | Shorts owe less; minting/arbitrage is rewarded. |
| `mark < index` | NF increases | Shorts owe more; longs are subsidized. |

Funding is applied lazily. A market can have no on-chain funding update for a period, then receive the full elapsed update during the next `modifyPosition()` or `liquidate()`.

## Funding Period

The funding period controls the speed of convergence. `StandardFundingModel` uses the market config `fundingPeriod`; if it is zero, the model falls back to `30 days`.

Shorter periods make mark-index divergence affect NF faster. Longer periods make funding smoother but allow persistent deviations to last longer.

## CDS Decay Funding

`CDSDecayFundingModel` intentionally does not use mark-index divergence. It reads `decayRateWad` from `MarketConfig`:

```text
F = decayRateWad
NF_new = NF_old * exp(-F * dt / secondsPerYear)
```

This represents deterministic premium decay for CDS-like markets. It reverts when `F = 0`, which makes the market configuration explicit.

This is a different economic model from the paper's standard rate-perp funding. Documentation and UI should present it as a CDS-specific funding model, not as the default RLP model.

## Mark And Spot Oracles

The standard funding model queries:

```text
markPrice = ISpotOracle(markOracle).getSpotPrice(positionToken, collateralToken)
indexPrice = IRLDOracle(rateOracle).getIndexPrice(underlyingPool, underlyingToken)
```

Liquidation also uses spot pricing when a `spotOracle` is configured. It chooses a conservative price for `indexPrice` in liquidation seize math:

```text
liquidationIndexPrice = min(rateIndexPrice, collateralSpotPrice)
```

This protects liquidators from overpaying in cases where collateral or index pricing diverges.

## Oracle Design Goals

The paper sets three oracle goals:

1. Use time-weighted average rates to reduce flash-spike liquidations.
2. Bound rate and price outputs so debt cannot become unbounded.
3. Validate reported rates against utilization and immutable IRM logic where possible.

The implementation already includes cap/floor behavior in `RLDAaveOracle` and pool observation integration through Ghost. TWAR and IRM validation remain the higher-assurance target for production oracle hardening.

## Integration Checklist

Before launching a market, verify:

| Check | Why it matters |
| --- | --- |
| `rateOracle` returns non-zero bounded index prices. | Funding and solvency require a valid index. |
| `markOracle` prices `positionToken` against `collateralToken`. | Standard funding queries this exact pair. |
| Broker oracle assumptions are satisfied. | Some broker paths expect both index and spot-style reads. |
| `fundingPeriod` is set deliberately. | It controls convergence speed. |
| CDS markets set `decayRateWad`. | `CDSDecayFundingModel` reverts when zero. |
| Spot oracle decimals and quote direction are audited. | Liquidation seizure depends on consistent units. |

## Failure Modes

| Failure | Effect |
| --- | --- |
| Zero mark or index price | Funding model reverts. |
| Stale or manipulated index | Debt values and liquidations drift from economic reality. |
| Wrong mark pair | Funding may revert or use invalid pricing. |
| Missing composite oracle where broker expects one | NAV and seize paths can fail. |
| Overly aggressive funding period | NF can move too quickly and increase liquidation risk. |
