The frontend is a Vite React app that combines simulation/indexer data with ethers-based transaction hooks. It is currently simulation-first for market state and uses direct contract calls for user actions.

## Data Sources

| Source | Files | Used for |
| --- | --- | --- |
| Simulation GraphQL | `frontend/src/hooks/useSimulation.js` | Market snapshot, pool state, index/mark candles, protocol stats, bonds, balances. |
| Envio / analytics GraphQL | `frontend/src/hooks/useMarketData.js`, `useEnvioStatus.js` | Lending data pages and header status. |
| RPC provider | `frontend/src/utils/provider.js` | Direct contract reads and transaction preparation. |
| Wallet signer | `frontend/src/utils/connection.js` | Ethers transaction signing. |

The main simulation hook polls global data, account data, and chart candles at separate intervals.

## User Transaction Hooks

| Hook | Contract calls |
| --- | --- |
| `useSwapExecution` | `BrokerRouter.executeLong`, `closeLong`, `executeShortWithMinOut`, `closeShort`; broker `operators`, `setOperator`. |
| `useBrokerAccount` | `PrimeBrokerFactory.createBroker`, `balanceOf`, token transfers. |
| `useBondExecution` | `BondFactory.mintBond`, `closeBond`. |
| `useTwammOrder` | Broker `submitTwammOrder`, `cancelTwammOrder`, claim flows, `setActiveTwammOrder`. |
| `usePoolLiquidity` | V4/POSM and broker LP operations. |
| `useBasisTradeExecution` | `BasisTradeFactory.openBasisTrade*`, `closeBasisTrade`. |
| `useSwapQuote` | V4 quote contract. |

The app uses ethers v6 and custom hooks. It does not use wagmi or viem for contract writes.

## Perp Terminal Flow

The main trading terminal centers on:

| Action | Protocol flow |
| --- | --- |
| Open long | Swap collateral into `wRLP` through `BrokerRouter.executeLong()`. |
| Close long | Swap `wRLP` back to collateral through `closeLong()`. |
| Open short | Mint debt through broker/core, then sell `wRLP` through router. |
| Close short | Spend collateral to buy `wRLP`, then burn to reduce debt. |
| Direct repay | Call broker `modifyPosition()` with negative debt when `wRLP` is already held. |

Before routed actions, the hook ensures `BrokerRouter` is an approved broker operator.

## Integration Contract Addresses

Market data is normalized into `marketInfo.infrastructure`, including:

| Field | Meaning |
| --- | --- |
| `broker_router` | Router for long/short execution. |
| `twamm_hook` | Hook address in pool key. |
| `twap_engine` | TWAMM execution engine. |
| `bond_factory` | Synthetic bond factory. |
| `basis_trade_factory` | Basis-trade factory. |
| `pool_manager` | Uniswap v4 pool manager. |
| `v4_quoter` | Quote helper. |
| `v4_position_manager` | Position manager for LP NFTs. |
| `permit2` | Permit2 address. |

Frontend transaction code should treat missing infrastructure fields as hard blockers, not as optional data.

## Validation Patterns

Current validation is mostly inline:

1. Required addresses are checked before transactions.
2. Wallet availability is checked before signing.
3. Broker router operator status is checked and set if missing.
4. Gas estimation is attempted with a buffer.
5. Revert signatures such as `SlippageExceeded` and panic underflow are formatted into user-readable errors.

For production, shared form validation should be added for collateral amounts, slippage, maturity, debt caps, min collateral ratios, and CDS-specific trigger/coverage constraints.
