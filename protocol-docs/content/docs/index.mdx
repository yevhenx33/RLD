RLD turns lending-market interest rates into programmable on-chain instruments. The protocol mints a fungible rate token, `wRLP`, whose index value is derived from a lending pool borrow rate:

```text
P_index = K * r_t
```

With `K = 100`, a 5% borrow rate maps to a 5.00 index price. That simple scalar lets users trade rates the same way they trade assets: longs buy `wRLP`, shorts mint it from collateral, and the market price is kept economically tied to the index through funding, arbitrage, and liquidation.

## What RLD Enables

| Product | User action | Protocol role |
| --- | --- | --- |
| Rate-level perps | Long or short borrow-rate exposure | Converts algorithmic interest rates into tradable spot liquidity. |
| Synthetic bonds | Short `wRLP` plus TWAMM unwind | Converts floating lending yield into programmable fixed-duration yield. |
| Fixed-rate borrowing | Long `wRLP` over loan duration | Offsets rising variable borrowing costs. |
| Rate volatility trades | Long or short rate beta | Expresses views on leverage demand and utilization shocks. |
| CDS markets | Long protection, short underwriting | Uses rate spikes and settlement rules as parametric solvency insurance. |

## Implementation Status

RLD is implemented as a smart-contract system with a simulation-first frontend and indexer. The core rate-perp, broker, funding, liquidation, market factory, bond, DEX, and TWAMM surfaces are present. CDS has concrete primitives in the contracts, including `settlementModule`, `globalSettlementTimestamp`, withdrawal queues, and `CDSDecayFundingModel`; the full production settlement module and live CDS UI execution remain integration surfaces to complete.
