This reference summarizes the authored protocol contracts and their responsibility boundaries. It excludes vendored dependencies under `contracts/lib`.

## Core

| Contract | Path | Purpose |
| --- | --- | --- |
| `RLDCore` | `contracts/src/rld/core/RLDCore.sol` | Singleton registry and accounting engine for all RLD markets. |
| `RLDStorage` | `contracts/src/rld/core/RLDStorage.sol` | Storage layout and transient-lock helpers used by `RLDCore`. |
| `RLDMarketFactory` | `contracts/src/rld/core/RLDMarketFactory.sol` | Deploys and registers markets. |
| `PrimeBrokerFactory` | `contracts/src/rld/core/PrimeBrokerFactory.sol` | Creates broker clones and mints broker ownership NFTs. |

## Account Layer

| Contract | Path | Purpose |
| --- | --- | --- |
| `PrimeBroker` | `contracts/src/rld/broker/PrimeBroker.sol` | Smart margin account owned by an NFT holder. |
| `PrimeBrokerOpsModule` | `contracts/src/rld/broker/PrimeBrokerOpsModule.sol` | Delegatecall module for larger broker operations. |
| `BrokerVerifier` | `contracts/src/rld/modules/verifier/BrokerVerifier.sol` | Validates whether an address is a broker from the expected factory. |

The broker is where user assets live. Core queries brokers but does not custody collateral.

## Tokens

| Contract | Path | Purpose |
| --- | --- | --- |
| `PositionToken` | `contracts/src/rld/tokens/PositionToken.sol` | ERC-20 `wRLP`, representing tokenized debt principal. |
| `WrappedAToken` | `contracts/src/shared/wrappers/WrappedAToken.sol` | Wrapper for interest-bearing collateral integrations. |

`PositionToken.mint()` and `burn()` are owner-gated, so deployment wiring must ensure the correct owner controls mint and burn rights.

## Funding

| Contract | Path | Model |
| --- | --- | --- |
| `StandardFundingModel` | `contracts/src/rld/modules/funding/StandardFundingModel.sol` | Mark-index funding: `NF *= exp(-fundingRate * dt / period)`. |
| `CDSDecayFundingModel` | `contracts/src/rld/modules/funding/CDSDecayFundingModel.sol` | Absolute decay: `NF *= exp(-F * dt / year)`. |

The standard model uses `markOracle`, `rateOracle`, `positionToken`, and `collateralToken`. The CDS model reads `decayRateWad` from market config and reverts if it is zero.

## Oracles

| Contract | Path | Purpose |
| --- | --- | --- |
| `RLDAaveOracle` | `contracts/src/rld/modules/oracles/RLDAaveOracle.sol` | Converts Aave current variable borrow rate to the RLD index price. |
| `GhostSingletonOracle` | `contracts/src/rld/modules/oracles/GhostSingletonOracle.sol` | Reads GhostRouter observations for mark or spot pricing. |
| `ChainlinkSpotOracle` | `contracts/src/rld/modules/oracles/ChainlinkSpotOracle.sol` | External spot collateral pricing. |

## Liquidation

| Contract | Path | Purpose |
| --- | --- | --- |
| `DutchLiquidationModule` | `contracts/src/rld/modules/liquidation/DutchLiquidationModule.sol` | Calculates health-based liquidation discount and collateral seizure. |

`liquidationParams` is packed:

| Bits | Field |
| --- | --- |
| `0..15` | Base discount in bps. |
| `16..31` | Maximum discount in bps. |
| `32..47` | Slope scaled by 100. |

## Access Control

| Permission | Holder |
| --- | --- |
| Create core market | `RLDCore.factory` only. |
| Create deployed market package | `RLDMarketFactory.owner` only. |
| Propose/cancel risk update | Per-market `curator`. |
| Activate global settlement | Per-market `settlementModule`. |
| Modify core position | Current `lock` holder only. |
| Liquidate | Permissionless if the target broker is below maintenance. |
| Broker action | Broker NFT owner or approved operator. |
| Broker seizure | `RLDCore` only. |
| Ghost market fees | Protocol owner or per-market fee controller. |

## Critical Invariants

1. A broker must end every lock solvent at the applicable ratio.
2. `wRLP` token supply is the source of truth for market total debt.
3. Core never trusts arbitrary users as brokers; verifier validation is required.
4. NAV reverts are treated as insolvency to avoid liquidation blocking.
5. Liquidation caches index price and NAV once to avoid inconsistent intra-call state.
6. CDS withdrawals are delayed when a settlement module is configured and the broker has debt.
7. V1 broker NAV only counts one active V4 position and one active TWAMM order.
