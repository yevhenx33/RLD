# Using RLD with AI

Every page on this site — documentation, contract references, and research papers — is machine-readable. Use any of the methods below to give your AI coding agent full context on the RLD Protocol.

---

## Quick Start

The fastest way to get your agent working with RLD is to point it at the documentation index:

```bash
claude -p "Read https://docs.rld.fi/llms.txt and summarize the RLD protocol"
```

---

## Read Any Page as Markdown

Every page on this site is available as plain Markdown. Append `/content.md` to the URL path under `/llms.mdx/`:

```text
https://docs.rld.fi/llms.mdx/content.md
https://docs.rld.fi/llms.mdx/traders/core-concepts/content.md
https://docs.rld.fi/llms.mdx/developers/smart-contracts/trading/content.md
https://docs.rld.fi/llms.mdx/research/parametric-cds/content.md
```

This returns raw Markdown content that can be pasted directly into any agent context window.

## llms.txt

Two index files are served at the root for bulk LLM consumption:

| URL | Contents |
| --- | --- |
| [`/llms.txt`](https://docs.rld.fi/llms.txt) | Concise index of all pages with titles, descriptions, and paths |
| [`/llms-full.txt`](https://docs.rld.fi/llms-full.txt) | Complete documentation content in a single file |

Use `/llms.txt` when you need the agent to understand the structure and pick relevant pages. Use `/llms-full.txt` when you want to load the entire knowledge base into context at once.

**Cursor / Windsurf:** Add `https://docs.rld.fi/llms.txt` as a documentation source in your editor's AI settings.

## MCP Server

The docs expose a [Model Context Protocol](https://modelcontextprotocol.io) server that gives agents programmatic access to search and retrieve documentation.

**Available tools:**

| Tool | Description |
| --- | --- |
| `list_pages` | List all documentation pages with titles, descriptions, and URLs |
| `get_page` | Retrieve the full Markdown content of any page by slug |
| `search_docs` | Search pages by keyword across titles and descriptions |

### Claude Code

```bash
claude mcp add --transport http rld-docs https://docs.rld.fi/api/mcp
```

### Manual Configuration

For Claude Desktop, Cursor, Windsurf, or any MCP-compatible client:

```json
{
  "mcpServers": {
    "rld-docs": {
      "url": "https://docs.rld.fi/api/mcp"
    }
  }
}
```

Once connected, the agent can navigate the full documentation tree — searching for topics, fetching specific pages, and reading contract interfaces without manual URL construction.

## Example Prompts

**Integration task:**
```text
Read https://docs.rld.fi/llms.txt to understand the RLD protocol. Then read 
the trading page at developers/smart-contracts/trading and write a TypeScript 
script that executes a long position using ethers.js v6.
```

**Research analysis:**
```text
Read the Parametric CDS whitepaper at research/parametric-cds and explain how 
the premium scaling mechanism ensures underwriters earn a convex risk premium 
that dominates passive supply yield.
```

**Architecture review:**
```text
Use the RLD docs MCP server to search for "TWAMM" and read all matching pages. 
Then explain how ghost accumulation and the epoch bitmap make accrual O(events) 
instead of O(intervals).
```
